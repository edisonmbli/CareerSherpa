5.1 核心原则：简历是“广告”，不是“编年史”

简历的唯一目的，是**精确匹配**对方的 JD 需求，证明你是“最佳人选”。它是一种“价值匹配”的广告，而不是你的“生平记叙”。`customize` 功能的核心，就是将用户的“生平记叙”（通用简历）重写为针对特定 JD 的“广告”（定制化简历）。

5.2 实践：如何量化成就 (X-Y-Z 法则)

这是 **最关键** 的实践方法。

- **法则**：(在 X 背景下) 通过 (Y 动作)，实现了 (Z 结果)。
- **X (Situation)**：你所处的背景或面临的问题。
- **Y (Action)**：你采取的关键动作（见 3.3）。
- **Z (Result)**：你的动作带来的**可量化**的结果。
- **LLM 任务**：LLM 在 `customize` 时，应尝试将用户平淡的描述改写为 X-Y-Z 格式。

5.3 实践：使用“动作动词” (Action Verbs)

使用强有力的动词，而不是被动或模糊的词汇。

- **弱动词 (Weak)**：负责、参与、协助、进行...
- **强动词 (Strong)**：**主导**、**推动**、**实现**、**优化**、**重构**、**设计**、**建立**、**将 A 提升至 B**...
- **LLM 任务**：LLM 应主动替换用户简历中的弱动词。

5.4 实践：如何根据 JD “打补丁” (Patching)

- **方法**：`customize` 任务必须利用 `match` 的分析结果。
- **步骤**：
1.  找出 JD 中的“必须项”关键词。
2.  确保这些关键词**显眼地**出现在定制化简历的“专业技能”和“项目描述”中。
3.  如果用户有相关经验但未提及该关键词，LLM 应尝试“转译”。（例如：JD 要求 "DevOps"，用户经验是 "负责 Jenkins 和 Docker"，LLM 应将其改写为 "负责基于 Jenkins 和 Docker 的 DevOps 流程"）。

5.5 案例库：“平淡”描述 vs “黄金”描述

**案例 1 (软件开发)**

- **平淡描述**：负责公司官网的日常维护和 bug 修复。
- **黄金描述 (XYZ + 动词)**：**主导** (Y) 了公司官网 (X) 的重构，通过优化组件和 API (Y)，将页面加载速度 (LCP) **提升了 40%** (Z)，并修复了 50 多个历史 bug (Z)。

**案例 2 (产品经理)**

- **平淡描述**：参与了新用户注册功能的设计。
- **黄金描述 (XYZ + 动词)**：**推动** (Y) 了新用户注册流程 (X) 的 A/B 测试，通过简化验证步骤 (Y)，将新用户**转化率提高了 15%** (Z)。

**案例 3 (市场营销)**

- **平淡描述**：负责公司的社交媒体账号运营。
- **黄金描述 (XYZ + 动词)**：**建立并运营** (Y) 了公司社交媒体矩阵 (X)，在 6 个月内实现粉丝**增长 120%** (Z)，单篇内容平均互动率**提升 30%** (Z)。
