## 原问题

```
当前在基础信息表单里，为社交链接提供的默认提示文案比较长，如果用户真的按这些规范来填写，会带来一些问题：长内容影响视觉排版效果，却没什么实际用途（无法点击）

针对这个场景，常见的处理方式是这样的？在简历中利用前缀icon+账号名，足够传递信息了吗？那么在编辑表单应作什么样的辅助提示呢？
```

## 优化方向

这是一个非常棒的洞察！你指出的痛点（长链接破坏排版且无实际意义）在很多简历工具中都普遍存在。

针对你的疑问，我的回答是：**是的，“Icon + ID/Username” 绝对是现代简历的最佳实践。**

以下是关于如何处理这个场景的完整交互设计方案，涵盖了**简历展示层**和**表单输入层**的优化策略。

---

### 1. 核心设计理念：展示与链接分离

传统的做法是“输入什么展示什么”，导致用户粘贴长链接时排版崩溃。
正确的做法应当是：**“输入的是 ID，展示的是 ID，点击的是链接”**。

- **视觉层 (UI):** 仅展示最关键的 `Username` 或短域名，保持整洁。
- **交互层 (UX):** 点击该文本/图标时，跳转到完整的 URL。
- **打印层 (Print):** 对于 GitHub/LinkedIn 等知名平台，HR 只需要看到 ID 就能搜索到，不需要把 `https://www...` 印出来。

---

### 2. 简历展示层 (Resume View) 优化建议

针对不同类型的社交链接，展示逻辑应有所区分：

#### A. 标准平台型 (GitHub, LinkedIn, Dribbble, Twitter/X)

- **展示样式：** `[Icon] username` (例如：GitHub 图标 + `edisonmbli`)
- **底层链接：** 自动拼接前缀 (例如：`https://github.com/edisonmbli`)
- **优势：** 极度节省横向空间，且因为有 Icon 存在，用户一眼就能识别是哪个平台。

#### B. 个人网站型 (Portfolio / Personal Website)

- **展示样式：** `[Icon] domain.com` (去除 `https://` 和 `www.`，例如：`edisonmbli.com`)
- **底层链接：** 用户输入的完整 URL。

---

### 3. 编辑表单层 (Edit Form) 优化建议

既然展示的是 ID，那么表单就应该引导用户**只输入 ID**，或者智能提取 ID。

#### 方案一：前缀引导法 (推荐 - 最直观)

利用 Input 组件的 `prefix` 或 `addon` 属性，把域名固定在输入框外面。

- **视觉效果：**
  `[ github.com/ ] [ 输入框: edisonmbli ]`
- **提示文案 (Placeholder):**
  不再展示链接，而是展示 ID 范例，例如 `your-id` 或 `edisonmbli`。
- **优点：** 用户一眼就知道只需要填后缀，不需要填 `https://`。

#### 方案二：智能清洗法 (容错性最强)

允许用户随便填（填 ID 或 完整链接均可），但由代码自动清洗。

- **输入框：** 普通文本框。
- **逻辑：** 监听 `onBlur` 或 `onChange`。如果检测到用户粘贴了 `https://github.com/edisonmbli`，自动通过正则提取出 `edisonmbli` 并回填到输入框，或者在存储时只存 ID。

---

### 4. 落地实施指南 (Actionable Guide)

#### Step 1: 优化 Input 组件 (Edit Form)

建议采用 **方案一 + 方案二** 的结合体。

**UI 示意：**

```text
GitHub
------------------------------------------------
| github.com/ |  edisonmbli                    |
------------------------------------------------
提示：请输入您的 GitHub 用户名

```

**代码逻辑 (Pseudo-code):**

```typescript
// 定义各个平台的配置
const SOCIAL_PLATFORMS = {
  github: {
    label: 'GitHub',
    domainDisplay: 'github.com/',
    urlPrefix: 'https://github.com/',
    placeholder: 'username', // 简短的提示
  },
  linkedin: {
    label: 'LinkedIn',
    domainDisplay: 'linkedin.com/in/',
    urlPrefix: 'https://www.linkedin.com/in/',
    placeholder: 'username',
  },
  website: {
    label: '个人网站',
    domainDisplay: 'https://', // 个人网站特殊处理
    urlPrefix: '', // 不拼接
    placeholder: 'your-portfolio.com',
  },
}

// 在表单渲染时
{
  Object.entries(SOCIAL_PLATFORMS).map(([key, config]) => (
    <div className="input-group">
      {/* 1. 视觉前缀，告诉用户这里不用填域名 */}
      <span className="input-prefix text-gray-400 bg-gray-50">
        {config.domainDisplay}
      </span>

      <input
        type="text"
        placeholder={config.placeholder}
        // 2. 智能清洗逻辑：允许用户粘贴完整链接，但自动提取ID
        onChange={(e) => {
          let val = e.target.value
          if (val.includes(config.domainDisplay)) {
            // 简单的清洗逻辑，移除前缀
            val = val
              .replace(/https?:\/\/(www\.)?/, '')
              .replace(config.domainDisplay, '')
          }
          updateFormData(key, val)
        }}
      />
    </div>
  ))
}
```

#### Step 2: 优化渲染组件 (Resume Template)

在简历模板中，通过简单的帮助函数来生成显示文本和链接。

```typescript
// utils/social-helper.ts

export const renderSocialItem = (platform: string, value: string) => {
  if (!value) return null

  let displayLabel = value
  let href = value

  switch (platform) {
    case 'github':
      displayLabel = value // 简历上只显示 "edisonmbli"
      href = `https://github.com/${value}`
      break
    case 'linkedin':
      displayLabel = value // 简历上只显示 "edisonmbli"
      href = `https://www.linkedin.com/in/${value}`
      break
    case 'website':
      // 去除协议头，只显示干净的域名
      displayLabel = value.replace(/^https?:\/\//, '').replace(/^www\./, '')
      href = value.startsWith('http') ? value : `https://${value}`
      break
  }

  return { displayLabel, href }
}

// 在 TemplateTechnical.tsx 中使用
const { displayLabel, href } = renderSocialItem('github', basics.github)
;<a href={href} className="flex items-center gap-1">
  <GithubIcon size={12} />
  <span>{displayLabel}</span>
</a>
```

### 总结

1. **简历展示：** 坚决采用 **`Icon + Username`**。对于 LinkedIn/GitHub，这已经是行业标准，完全足够传递信息且足够美观。
2. **表单提示：** \* **Placeholders:** 改短，使用 `username` 或 `edisonmbli`。

- **Prefix UI:** 最好在输入框前显示灰色的 `github.com/`，给予用户强烈的心理暗示“**这里只要填名字**”。
