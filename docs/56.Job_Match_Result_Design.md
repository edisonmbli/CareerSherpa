# 56. Job Match Result Design System (V6 Standard)

> **Status**: Final / Standardized
> **Last Updated**: 2026-01-15 (Reflecting V14 Implementation)
> **Scope**: Global Design Language for "Report" & "Analysis" Pages

## 1. Design Philosophy: "The Editorial Report"

We have evolved the interface from a "SaaS Dashboard" to a **"High-End Magazine Feature"**. The goal is to make complex AI analysis feel like a curated, human-written consulting report.

-   **Visual Metaphor**: _The Editor's Desk_. Clean paper, highlighted key texts, margin notes, and strong typographic hierarchy.
-   **Core Attribute**: **Tangibility**. Use texture, layering, and "ink" colors (Emerald/Amber/Rose) rather than digital neons.
-   **Key Principle**: **Vertical Continuity**. A strong vertical thread (Keyline) connects all sections, driving the reader down the page.

---

## 2. Visual Foundation

### 2.1 Typography System

We use a tri-font system to separate **Narrative**, **UI**, and **Data**.

| Role | Font Family | Tailwind Class | Usage |
| :--- | :--- | :--- | :--- |
| **Editorial Headings** | **Playfair Display** | `font-serif font-bold tracking-tight` | Section titles, Hero headers. |
| **Narrative Body** | **Inter** | `font-sans text-sm leading-relaxed` | Main analysis text, paragraphs. |
| **Data & Tags** | **JetBrains Mono** | `font-mono font-bold tracking-widest` | Scores, UI Labels (HOOK/VALUE), decorative numbers. |

### 2.2 Color System (The Match Score Tiers)

The entire page theme adapts to the match score.

| Tier | Score Range | Color Token | Semantic Feel |
| :--- | :--- | :--- | :--- |
| **Exceptional** | **85 - 100** | `Emerald` | Success, Growth, Profit. |
| **Good / Solid** | **60 - 84** | `Amber` | Caution, Opportunity, Warmth. |
| **Risky** | **0 - 59** | `Rose` | Critical, Urgent, Stop. |

> **Dark Mode Rule**: In dark mode, shift lighter/softer (`400`/`500` weights) to avoid harsh contrast. Use `bg-slate-950` as the canvas.

## 3. Core UI Components

### 3.1 The "Half-Highlight" Header (Signature Style)

Instead of distinct "Cards" with headers, we use **Offset Highlights** to mark sections.

-   **Visual**: A solid block of color (`bg-{theme}-200`) sitting _behind_ and _misaligned_ with the serif title.
-   **Implementation**:
    ```tsx
    <div className="relative inline-block">
      {/* The Highlight Block */}
      <div className="absolute bottom-4 -left-4 w-24 h-5 -z-10 bg-emerald-200 dark:bg-emerald-900/50" />
      {/* The Text */}
      <span className="font-serif text-2xl font-bold relative z-10">Title</span>
    </div>
    ```

### 3.2 The Continuous Keyline (Layout)

A single vertical or dotted line runs through the entire content column, acting as the spine.

-   **Structure**: `absolute top-[-24px] bottom-[-24px] w-px bg-slate-200`.
-   **Placement**: 12px-24px from the left edge.
-   **Bullets**: Hollow circles (`w-1.5 h-1.5 bg-white border-2`) placed _on top_ of the line.

---

## 4. Component Patterns

### 4.1 "Expert Verdict" (The Quote Card)
A distinct block for the AI's high-level summary.
-   **Background**: `bg-stone-50` (Warm/Paper) to contrast with the cool `bg-slate-50` page.
-   **Watermark**: A large, low-opacity Icon (`Quote` or `Sparkles`) rotated in the top-left corner (`opacity-5`).
-   **Drop Cap**: The first letter is large, serif, and bold to mimic editorial layouts.

### 4.2 Analysis Lists (Accordion)
-   **Grid**: `grid-cols-[24px_1fr]` (Gutter + Content).
-   **Points**: `ReactMarkdown` with `p` as inline and `strong` as **Wavy Underline** (`decoration-wavy underline-offset-2`).
-   **Tips**: Sidebar style. Little icons (`FileText`, `Mic`) with uppercase labels to distinguish actionable advice from analysis.

### 4.3 Smart Pitch
-   **Tags**: purely text-based. No chips/pills.
    -   Style: `text-[9px] font-bold uppercase tracking-widest`.
    -   Accent: A tiny `w-1.5 h-1.5` dot of the theme color preceding the tag.
-   **Script**: Monospace or clean Sans, rendered as a script.
-   **Mobile**: A static legend footer (H - Hook, V - Value) instead of hover tooltips.

---

## 5. Mobile Responsiveness

-   **Strategy**: "Drawer First". The report often lives in a Drawer on mobile.
-   **Typography**: Scale Titles down (2xl -> xl).
-   **Spacing**: Reduce vertical gaps (`space-y-8` -> `space-y-6`).
-   **Interactions**: Replace Hover interactions (like "Click to Copy" tooltips) with static helper text or legends.

## 6. Checklist for New Pages

When applying this design to new pages (e.g. Interview Prep), check:
1.  [ ] Are titles using **Playfair Display** with the **Half-Highlight**?
2.  [ ] Is there a **Vertical Keyline** connecting the items?
3.  [ ] Are you using **Stone-50** for specific "Human/AI Voice" containers?
4.  [ ] Are tags **Text-only** (editorial) vs Chips (SaaS)?
5.  [ ] Is `ReactMarkdown` styled with **Wavy Underlines** for emphasis?
