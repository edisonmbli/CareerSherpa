# AI 简历助手视觉与体验升级方案 (Revamp v2.0)

> **核心目标**：打造一位“高智商、有温度、知进退”的简历教练。
> **设计原则**：
>
> 1.  **闪耀但不喧宾夺主 (Shine but don't Shout)**：AI 面板是辅助角色，应像“副驾驶”一样，在用户需要时提供高价值信息，而不是抢夺“驾驶员”（画布编辑区）的注意力。
> 2.  **跨端一致性 (Seamless Adaptation)**：在桌面端提供沉浸式侧边栏体验，在移动端提供原生级的抽屉（Drawer）体验。

---

## 🎨 桌面端设计：平衡“惊艳感”与“沉浸感”

### 1. 视觉氛围：从“炫光”回归“微光” (Ambient Glow)

为了避免抢戏，我们将放弃大面积的渐变背景，转而使用**“环境光”**来营造智能感。

- **背景策略**：
  - 保持背景为纯净的 `bg-white` (Dark: `bg-zinc-900`)，以维持与主界面的统一性。
  - **顶部微光**：仅在 Header 区域下方投射一道极淡的靛蓝/紫色光晕（使用 CSS `box-shadow` 或 绝对定位的 `blur` 元素），暗示 AI 的存在，但不干扰阅读。
  - **边界呼吸**：面板左侧边缘（与画布交界处）使用 `border-l border-indigo-100`，在 AI 内容弹出时，边框可以有一道微弱的光流闪过，随后静止。

### 2. 头部区域：克制而专业的身份标识

Header 不应过高，以免挤压内容区域。

- **布局优化**：
  - **左侧**：保留 `Sparkles` 或 `Bot` 图标，但缩小尺寸（w-4 h-4），使用品牌色（Indigo-600）。
  - **标题**：主标题“优化建议”使用 `text-sm font-semibold`，副标题（如“基于 JD 分析”）使用 `text-xs text-muted-foreground`。
  - **右侧**：除了关闭按钮，增加一个**“收起/展开”**的微交互按钮（如果空间允许），或者一个简单的**“复制全部”**图标按钮，提升工具属性。

### 3. 内容呈现：结构化卡片流 (Structured Cards)

这是“闪耀”的核心。通过排版和微小的容器设计来提升价值感，而不是靠颜色堆砌。

- **卡片设计**：
  - **无边框风格**：为了减少视觉噪点，卡片可以不加边框，而是使用极淡的背景色块 `bg-slate-50` (Dark: `bg-white/5`)。
  - **左侧色条 (Accent Line)**：每张卡片左侧使用 `3px` 宽的色条来区分类型（蓝色=调整，紫色=理由），这比大面积色块更优雅、更不抢眼。
- **排版层级**：
  - **标题**：使用深色文字，字重 `font-medium`。
  - **正文**：调整行高至 `leading-relaxed`，确保长时间阅读不累。
  - **关键词**：使用底色高亮（Marker effect）代替简单的颜色变化，模拟“批改作业”的真实感。

## 📱 移动端设计：原生级体验 (Native-like Experience)

移动端屏幕空间寸土寸金，侧边栏（Side Panel）模式在手机上体验极差。我们需要彻底重构为**底部抽屉（Bottom Sheet / Drawer）**模式。

### 1. 交互形态：Bottom Drawer

- **触发**：点击底部的“AI 建议”浮动按钮（FAB）或工具栏图标。
- **容器**：一个从底部滑出的面板，覆盖屏幕高度的 **85%** 或 **50%**（支持手势拖拽改变高度）。
- **手势**：支持**下滑关闭**，这是移动端用户最自然的肌肉记忆。

### 2. 移动端 Header

- **把手 (Grabber)**：顶部中心必须有一个灰色的圆角短横条，暗示可拖拽。
- **极简标题**：仅保留“AI 建议”标题和关闭按钮，去除副标题以节省纵向空间。
- **吸顶效果**：当内容滚动时，Header 应该有一层 `backdrop-blur` 效果，防止文字重叠。

### 3. 移动端内容适配

- **卡片堆叠**：
  - 卡片宽度占满屏幕（左右留 `padding-4`）。
  - 增大字号：正文最小 `14px`，标题 `16px`，防止眼疲劳。
  - 增大点击热区：如果卡片有交互（如“复制”），按钮尺寸至少 `44x44px`。
- **操作栏 (Sticky Footer)**：
  - 如果在移动端修改简历不便，可以增加一个**“一键应用”**或**“复制到剪贴板”**的悬浮按钮在 Drawer 底部，方便用户直接拿走结果。

## � 技术实现建议 (Tailwind CSS)

### 桌面端微光效果

```tsx
<div className="absolute top-0 left-0 right-0 h-32 bg-gradient-to-b from-indigo-50/50 to-transparent pointer-events-none" />
```

### 卡片样式 (优雅版)

```tsx
<div className="group relative pl-4 pr-3 py-3 bg-slate-50/50 hover:bg-slate-50 rounded-r-lg transition-colors">
  {/* 左侧色条 */}
  <div className="absolute left-0 top-3 bottom-3 w-[3px] bg-indigo-500 rounded-full" />
  {/* 内容 */}
  <h3 className="text-sm font-medium text-slate-900 mb-1">...</h3>
  <p className="text-xs text-slate-600 leading-relaxed">...</p>
</div>
```

### 移动端 Drawer (使用 Vaul 或类似库)

建议引入 `vaul` (Radix UI 团队开发的 Drawer 组件) 来实现丝滑的拖拽效果，而不是自己手写 CSS transform。

```tsx
// 伪代码
<Drawer.Root>
  <Drawer.Trigger>AI 建议</Drawer.Trigger>
  <Drawer.Portal>
    <Drawer.Content className="bg-white flex flex-col rounded-t-[10px] h-[85vh] mt-24 fixed bottom-0 left-0 right-0">
      <div className="mx-auto w-12 h-1.5 flex-shrink-0 rounded-full bg-zinc-300 mb-8 mt-4" />{' '}
      {/* Handle */}
      {/* Content */}
    </Drawer.Content>
  </Drawer.Portal>
</Drawer.Root>
```
