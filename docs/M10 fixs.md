1）✅ 切换模板后，其默认的“出厂设置”并没有被正常加载
- 这里“出厂设置”，指的是每个模板底部，对外export的默认config
- 测试的体感是，切换模板后，样式中没有显示当前已加载的字体、间距等细节是什么，让人困惑。我自己肉眼比对了下，基本确认默认值是没有成功加载的（譬如，切换technicak模板后，并没有自动应用JetBrains Mono字体；切换Corporate模板后，自动加载的是Roboto，而不是默认配置的Noto Serif / Song；等等）
- 请重点排查下问题根因，看是否因为历史遗留的硬编码还在生效，导致新开发的export默认值机制实际上没有派上用场？
- 也请重点检查字体映射机制： 确保字体组件（比如 FontSelect）中、以及默认配置的export参数值，其value 值（如 "JetBrains Mono"）能正确映射到 CSS 的 font-family 栈。例如： fontFamily: "JetBrains Mono" 对应的 CSS 应该是 font-family: 'JetBrains Mono', monospace;
- 为了增强用户感知： 当用户切换模板时，可以做一个微小的 Toast 提示：“已应用最适合该模板的字体与配色”，增加产品的体贴感。但需要注意，样式上需要与当前的页面style保持平衡、协调
