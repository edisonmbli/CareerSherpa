# Dark Mode UI Enhancement Plan

## 1. Core Design Philosophy
To achieve a "Premium" and "Professional" look (similar to Vercel/Linear/Stripe), we must move away from the current "Developer Tool" aesthetic (Blue/Slate heavy) to a more "Industrial" and "Physical" design system.

### 1.1. Color Palette: Shift to Zinc
*   **Current**: Slate (Hue 260, Chroma ~0.02). Too blue/cold.
*   **Target**: Zinc (Hue 240, Chroma < 0.01). Deeper, more neutral, less "digital".
*   **Rationale**: Zinc provides a high-end, neutral canvas that allows content (text/images) to pop. It avoids the "cheap monitor" blue tint often associated with default dark modes.

### 1.2. Elevation Modeling (Physics of Light)
*   **Concept**: In a dark environment, surfaces closer to the light source (user) must be **lighter**.
*   **Hierarchy**:
    1.  **Background (Lowest)**: Darkest (Deep Zinc).
    2.  **Surface/Card (Mid)**: Lighter than background.
    3.  **Overlay/Dialog (Highest)**: Lightest surface.
    4.  **Border (Definition)**: Used to define edges when contrast ratios are low.

## 2. Audit Findings & Issues

### 2.1. The "Valley Effect" (Inverted Depth)
*   **Location**: `app/[locale]/profile/page.tsx` and **`app/[locale]/workbench/layout.tsx`**
*   **Issue**: A generic container uses `bg-muted/60` (Light) which wraps `AppCard` components (Darker `bg-card`).
*   **Result**: The cards appear to be "sinking" into the container rather than floating on top.
*   **fix**: Remove the `bg-muted/60` wrapper background. Cards should sit directly on the page background, or the wrapper should be transparent with a border if grouping is needed.

### 2.2. The "Black Hole" Dialogs
*   **Location**: `components/ui/dialog.tsx` (Line 28), `components/ui/sheet.tsx` (Line 61)
*   **Issue**: These components hardcode `bg-background` (Level 0 depth).
*   **Result**: When a Dialog opens over a Card (Level 1 depth), logic dictates it should be brighter. Instead, it is darker, creating a visual hole.
*   **Fix**: Update these components to use `bg-popover` (Level 2 depth) or `bg-card` (Level 1) with a distinct border.

### 2.3. Palette Inconsistencies
*   **Location**: `app/globals.css`
*   **Issue**: Manual usage of Hue 260 and distinct chroma values.
*   **Fix**: Standardize on Hue 240 (Zinc) with minimal chroma (0.005 - 0.01).

## 3. Implementation Specifications

### 3.1. New Color Tokens (Action: Update `app/globals.css`)

**Base Dark Palette (Zinc)**
```css
  --background: oklch(0.12 0.006 240);  /* Deepest base */
  --card: oklch(0.145 0.006 240);       /* Surface Level 1 */
  --popover: oklch(0.145 0.006 240);    /* Surface Level 2 (Same as card or slightly lighter oklch(0.16...)) */
  --muted: oklch(0.145 0.006 240);      /* Secondary surface */
  --border: oklch(0.20 0.006 240);      /* Subtle definition */
```
*Note: We reduce chroma to 0.006 to keep it very neutral but not "dead" gray.*
